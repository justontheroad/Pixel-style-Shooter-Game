# 像素射击游戏 — 后续迭代方案 (v3.x 大版本路线图)

> 本文档为后续迭代的整体路线规划，覆盖 v3.0 / v3.5 / v4.0 三个大版本。
> 制定依据：项目当前状态分析（v2.5 已落地）+ 用户决策（体验打磨 → 内容扩展 → 长线留存；纳入 Boss；仅功能迭代不清技术债）。

---

## 1. 总览

### 1.1 路线图一览

| 版本 | 主题 | 关键交付 | 里程碑定位 |
|------|------|---------|----------|
| **v3.0** | 体验打磨（Polish） | 暂停系统、横屏提示、城市废墟背景、角色形象升级、临时武器闪烁预警、连击/拾取/受击手感强化 | 提升「上手即沉浸」的精致度 |
| **v3.5** | 内容扩展 + Boss 机制 | 3 个 Boss 战、第 15-16 阶段（深渊期 / 终焉期）、4 把新武器、3 类新障碍、2 种新道具、2 套场景主题 | 大幅延长可玩时长、引入关卡高潮 |
| **v4.0** | 长线 / 留存系统 | 成就系统、武器解锁/收藏、每日任务、本地排行榜、角色皮肤、复活机制升级 | 建立回访动机与目标感 |

### 1.2 设计原则

- **不破坏现有 v2.5 平衡**：所有新内容以「叠加」方式加入；不下调现有数值
- **保持像素美学一致**：新增视觉元素沿用 240×400 低分辨率、6 色板风格
- **零构建依赖**：继续使用 ES Modules + importmap，不引入打包工具
- **数据驱动**：新内容（Boss、皮肤、成就）继续以 config.js 数组扩展
- **不在本轮处理技术债**：sharedPixelGeo、DIFFICULTY_STAGES 双源真相等问题留至 v4.x

## 2. v3.0 — 体验打磨（Polish）

**主题**：让玩家在前 30 秒就感受到「精致」。

### 2.1 暂停系统

- **目的**：解决 `state.paused` 字段定义却未使用的体验缺失；玩家随时可暂停查看 HUD / 休息
- **改动文件**：
  - `js/input.js`：监听 `KeyP` / `Escape` / 移动端长按
  - `js/game.js`：`gameLoop` 检测 `state.paused` 时跳过 update（仍 render）
  - `js/ui.js`：新增暂停遮罩（半透明黑底 + 「PAUSED」+ 「继续 / 重新开始 / 退出」按钮）
  - `index.html` + `css/style.css`：暂停按钮 DOM
- **交互**：暂停时背景音乐降速 50%，恢复时回正
- **验收**：暂停期间所有计时器（道具/波次/连击）停止；恢复后无跳帧

### 2.2 横屏旋转提示

- **目的**：PRD 4.3 已要求但未实现；移动端横屏时游戏被压扁
- **改动文件**：
  - `js/main.js`：`window.matchMedia('(orientation: landscape)')` 监听
  - `index.html`：新增 `#orientation-mask` DOM（手机旋转图标 + 文字）
  - `css/style.css`：横屏时显示遮罩并暂停游戏
- **验收**：在 Chrome DevTools 移动模拟横屏切换无误

### 2.3 城市废墟背景

- **目的**：策划方案明确要求「深色城市废墟，缓慢滚动增加纵深感」，当前仅星空
- **改动文件**：
  - `js/scene.js`：新增 `createCityBackground()`，用 Three.js Sprites 或 PlaneMesh 绘制远景剪影（破损建筑、天线、月亮）
  - 视差滚动：前景速度 1.0×、中景 0.5×、远景 0.2×
  - 与星空 Points 共存（星空在最远层）
- **资源**：纯程序化生成（无外部贴图），用 Canvas2D 绘制后转纹理
- **验收**：FPS 不下降 5% 以上；移动端可选关闭

### 2.4 角色形象升级

- **目的**：策划方案要求「随武器升级更换持枪姿态 / 多样化枪支像素图」，当前仅尺寸+颜色变化
- **改动文件**：
  - `js/player.js`：`updatePlayerVisuals()` 内为 12 种武器各绘制独立的 Canvas2D 像素图（8×8 ~ 16×16）转纹理
  - 武器图素材规范：左轮短管、加特林粗管+弹链、火箭筒含弹头、激光炮含散热片
  - 角色身体保持不变（沿用现有像素小人）
- **验收**：每级武器视觉可辨识；切换瞬间无白屏

### 2.5 临时武器到期预警

- **目的**：v2.5-plan 已规划但仅完成分身预警，临时武器未做闪烁
- **改动文件**：
  - `js/items.js`：`tempWeaponTimer < 3` 时设置 `state.tempWeaponWarning = true`
  - `js/player.js`：`updatePlayerVisuals()` 检测预警标志，让 gunMesh 每 0.2s 闪烁
  - `js/ui.js`：HUD 临时武器计时器最后 3 秒变红色
- **验收**：预警与现有分身预警节奏一致

### 2.6 手感反馈强化

- **目的**：放大「击碎瞬间」的爽快感
- **改动文件**：
  - `js/effects.js`：
    - 命中粒子数量随武器等级增加（1-4 级:3 粒，5-8:5 粒，9-12:8 粒）
    - 击碎特效新增「径向冲击波」环（0.2s 扩散并淡出）
    - 连击 ×10/×20/×30 时屏幕边缘出现金色闪光 vignette
  - `js/audio.js`：连击 ×30+ 时主音乐叠加和声层
  - `js/game.js`：摧毁 10 级障碍（钢铁堡垒）时屏幕震动幅度 +50%
- **验收**：高连击状态下视觉层次明显加深，但不影响识别障碍物

### 2.7 v3.0 交付清单

| 子任务 | 涉及文件 | 工作量估算 |

| 临时武器闪烁预警 | items.js, player.js, ui.js | S |

## 3. v3.5 — 内容扩展 + Boss 机制

**主题**：从「无限刷怪」升级到「有起伏的关卡体验」。

### 3.1 Boss 系统（核心机制）

#### 3.1.1 触发与流程

- 玩家通关每个阶段的最后一波（waves.js 中每个 stage 的末尾），有几率触发该阶段对应 Boss
- 触发概率：阶段 4 起 20%，每阶段递增，阶段 10+ 必触发 Boss
- 流程：波次结束 → 屏幕震动 + 「WARNING」横幅 → Boss 从上方降落
- Boss 期间禁止普通障碍生成，禁止切换通道（或可切换但 Boss 横跨双通道）

#### 3.1.2 Boss 数据结构（新增 `js/bosses.js`）

```javascript
export const BOSSES = [
  {
    id: 'tank',          // 装甲坦克 (对应 阶段 4-5)
    name: '装甲坦克',
    hp: 3000,
    width: 80,
    phases: [
      { hpRatio: 1.0, attack: 'cannon', interval: 1.5 },
      { hpRatio: 0.5, attack: 'mgSpread', interval: 0.8 }, // 半血激怒
    ],
    weakPoint: { x: 0, y: 20, radius: 8, dmgMult: 2.0 }, // 弱点 2x 伤害
    reward: { weaponLevel: '+2', score: 5000 }
  },
  {
    id: 'mech',          // 机甲战士 (对应 阶段 8-9)
    name: '机甲战士',
    hp: 6000,
    attacks: ['homingMissile', 'laserSweep', 'shieldBash'],
    // ...
  },
  {
    id: 'overlord',      // 钢铁霸主 (对应 阶段 12-14)
    name: '钢铁霸主',
    hp: 12000,
    attacks: ['orbitalStrike', 'cloneSummon', 'beamCannon'],
    // ...
  }
];
```

#### 3.1.3 Boss 攻击模式

| 攻击 | 描述 | 玩家应对 |
|------|------|---------|
| `cannon` | 单发大弹丸，0.5s 蓄力红色预警 | 切换通道 |
| `mgSpread` | 三向散弹 | 切到无弹通道 |
| `homingMissile` | 慢速追踪弹，可被击毁 | 击落 / 利用障碍卡位 |
| `laserSweep` | 横扫光束，0.8s 蓄力 | 切到非扫描通道 |
| `shieldBash` | 冲撞玩家所在通道 | 切换通道 |
| `orbitalStrike` | 屏幕落点预警 + 延迟爆炸 | 避开落点 |
| `cloneSummon` | 召唤 2 个小怪 | 优先清小怪 |
| `beamCannon` | 全屏蓄能光束 | 必须切换通道 |

#### 3.1.4 改动文件

- `js/bosses.js`：**新文件**，Boss 数据 + spawn/update/destroy 逻辑
- `js/waves.js`：阶段末尾插入 Boss 触发钩子
- `js/state.js`：新增 `state.activeBoss`、`state.bossPhase`
- `js/weapons.js`：子弹碰撞检测扩展到 Boss 与弱点
- `js/effects.js`：弱点高亮、Boss 血条（屏幕顶部横条）、蓄力预警
- `js/ui.js`：Boss 名称横幅、Boss 血条
- `js/audio.js`：Boss 战 BGM 切换 + 蓄力警报音 + Boss 受击音

#### 3.1.5 验收

- 三个 Boss 可正常触发、攻击、被击败
- Boss 战不会因子弹池满（MAX_BULLETS=30）导致僵局
- Boss 死亡后正常进入下一阶段，奖励武器升级生效

### 3.2 新增阶段（深渊期 / 终焉期）

- **目的**：扩展 14 阶段到 16 阶段，配合 Boss 形成「Boss 战 → 缓冲 → 更强 Boss」节奏
- 改动 `js/waves.js`：新增 `waveDefs[14]`、`waveDefs[15]`
- 难度：障碍物全 9-10 级，速度倍率 1.4-1.6，必触发钢铁霸主 Boss

### 3.5 新道具（2 种）

- **未来都市**：霓虹紫蓝背景，电网粒子，障碍物色调偏青

## 4. v4.0 — 长线 / 留存系统

**主题**：让玩家「想再玩一次」。

### 4.1 成就系统

- **改动文件**：`js/achievements.js`（新文件）、`js/state.js`、`js/ui.js`
- 30 个成就（如：「初杀」「连击 100」「击败钢铁霸主」「不死通关阶段 10」「集齐 12 把武器」）

- 触发即弹出右上角 toast（金色边框 + 解锁音）

### 4.2 武器解锁 / 收藏

- 首次拾取过的武器解锁到「武器图鉴」
- 主菜单新增「图鉴」入口，展示已解锁武器图标 + 数据
- 未解锁武器显示为问号轮廓
- 改动文件：`js/ui.js`、`js/state.js`、新增 `index.html` 图鉴弹层

### 4.3 每日任务

- 每天 3 个随机任务（如：「击碎 100 个障碍」「使用激光炮造成 1000 伤害」「不使用护盾通过阶段 5」）

- 改动文件：`js/dailyTasks.js`（新）、`js/ui.js`

### 4.4 本地排行榜

### 4.5 角色皮肤

- 改动文件：`js/player.js`、`js/config.js`、`js/ui.js`

### 4.6 复活机制升级

## 6. 假设与决策（Assumptions & Decisions）

| # | 假设 / 决策 | 依据 |
|---|------------|------|
| A1 | 沿用 ES Modules + importmap，不引入 React/Vue 等 UI 框架 | 当前架构纯原生，引入框架成本过高 |
| A2 | 所有新视觉资源继续程序化生成（Canvas2D / Three.js Geometry） | 项目零外部资源依赖原则 |
| A3 | Boss 阶段不允许暂停（与「紧张感」冲突） | 单机游戏惯例 |
| A4 | 每日任务使用本地日期，不联网校验 | 本游戏无服务端 |
| A5 | 金币系统仅本地存储，不接通付费 | PRD 未提到内购 |
| A6 | 「不在本轮处理技术债」：sharedPixelGeo、DIFFICULTY_STAGES 双源真相、固定时间步物理 — 留待 v4.x 后处理 | 用户决策 |
| A7 | 横屏遮罩沿用游戏暂停机制（暂停 + 黑屏） | 复用 v3.0 暂停系统 |
| A8 | 新武器（13-16 级）只在 v3.5 新阶段掉落，不影响早期平衡 | 不破坏 v2.5 平衡原则 |
