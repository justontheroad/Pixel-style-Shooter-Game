# 像素风格射击游戏 — 完整游戏策划方案

> Pixel-style Shooter Game — Full Game Design Document

---

## 一、游戏概述

| 项目 | 内容 |
|------|------|
| 游戏名称 | 像素风格射击游戏（Pixel-style Shooter Game） |
| 游戏类型 | 竖屏无限射击 / 休闲动作 |
| 目标平台 | H5 Web（PC + 移动端），适配竖屏操作 |
| 核心体验 | 快节奏射击 + 躲避 + 升级，像素风格的视觉反馈带来持续的爽快感 |

**游戏简介：**

玩家操控像素角色在屏幕下方左右两个通道内移动，持续向上射击。障碍物从屏幕上方不断滚落，玩家需要击碎障碍物获取积分。障碍物到达屏幕底部或触碰角色则游戏失败。随着游戏推进，难度逐步提升，同时出现更高级的枪支道具供拾取，让玩家在挑战中不断获得升级的快感和视觉满足。

## 二、核心玩法机制

### 2.1 场景布局

```
  ┌─────────────────────┐
  │     ↓ 障碍物生成区    │  障碍物在此区域随机生成并开始下落
  │                     │
  │   ┌───┐     ┌───┐   │  障碍物分布在左右两条通道中
  │   │ ■ │     │ ■ │   │
  │   └───┘     └───┘   │
  │                     │
  │  ║  左通道  ║  右通道 ║  两条垂直通道，中间有分隔带
  │  ║         ║        ║
  │  ║    🔫   ║   🔫   │  角色在通道底部活动
  └─────────────────────┘
```

- 屏幕被中央分隔带分为左右两条通道
- 角色只能在所在通道内左右移动（不可跨通道）
- 障碍物在两条通道中随机生成，沿通道向下滚动
- 子弹沿通道向上飞行，击中同通道的障碍物

### 2.2 角色控制

**核心操作：点击切换通道**

| 平台 | 操作方式 | 行为 |
|------|---------|------|
| PC端 | 鼠标点击画面任意位置 | 角色切换到另一通道 |
| 移动端 | 手指触屏点击画面任意位置 | 角色切换到另一通道 |

- 角色始终在两个通道之间切换，点击一次切换到对面通道
- 角色自动持续射击，无需手动开火
- 角色切换通道时有平滑过渡动画（约0.15秒）
- 操作极简：只需"点击"一个动作，降低上手门槛

### 2.3 射击机制

- 角色自动向正上方持续射击
- 射速由当前枪支决定
- 子弹沿直线向上飞行
- 子弹命中障碍物时：
  - 扣除障碍物对应生命值
  - 触发命中粒子特效
  - 触发命中音效
  - 显示伤害数字飘字（-XX）
- 部分枪支有特殊弹道（见枪支系统）

### 2.4 障碍物机制

- 障碍物从屏幕顶部随机位置生成
- 以恒定速度向下滚动（速度随难度提升）
- 障碍物显示当前生命值条
- 生命值归零时：
  - 播放炸开粒子特效
  - 播放爆炸音效
  - 障碍物消失
  - 获得对应积分
- 障碍物到达屏幕底部底线 或 碰撞角色：
  - 游戏结束

### 2.5 积分系统

**积分来源：**

- 击碎障碍物：基础积分 = 障碍物等级 × 10
- 连击加成：连续击碎障碍物不漏过，连击倍率递增

| 连击数 | 倍率 |
|--------|------|
| 5连击 | 1.5x |
| 10连击 | 2.0x |
| 20连击 | 3.0x |
| 50连击 | 5.0x |

- 连击在漏过障碍物（障碍物到达底部）时重置为0

### 2.6 游戏流程

```
开始界面 → 点击开始 → 游戏进行（难度递增）→ 游戏失败 → 结算界面
                                                    ↓
                                               显示得分/最高分
                                               重新开始 / 返回
```

- 游戏初始装备1级左轮手枪
- 2-12级枪支通过拾取道具获得
- 拾取后立即替换当前枪支
- 被替换的枪支不保留（简化设计，鼓励策略选择）

## 四、障碍物系统（1-10级）

### 4.1 障碍物属性表

| 等级 | 名称 | 生命值 | 像素尺寸 | 下落速度 | 积分 | 外观描述 |
|------|------|--------|----------|----------|------|----------|
| 1 | 木箱 | 20 | 24×24 | 1.0 | 10 | 棕色木纹方块 |
| 2 | 树干 | 40 | 20×32 | 0.9 | 20 | 棕色竖向圆木 |
| 3 | 石头 | 80 | 28×28 | 0.8 | 30 | 灰色不规则石块 |
| 4 | 铁桶 | 130 | 22×28 | 0.85 | 40 | 深灰色圆柱桶 |
| 5 | 沙袋 | 200 | 30×20 | 0.7 | 50 | 土黄色长条沙袋 |
| 6 | 铁板 | 300 | 32×24 | 0.75 | 60 | 银灰色金属板 |
| 7 | 水泥墙 | 450 | 36×36 | 0.6 | 70 | 浅灰色方块墙 |
| 8 | 汽车 | 700 | 40×28 | 0.5 | 80 | 彩色像素小车 |
| 9 | 装甲车 | 1100 | 44×32 | 0.45 | 90 | 军绿色装甲车 |
| 10 | 钢铁堡垒 | 1800 | 48×48 | 0.35 | 100 | 深灰色大型堡垒 |

### 4.2 障碍物特性说明

- **木箱**：最脆弱，1级枪支即可轻松击碎，入门级障碍
- **树干**：细长形状，占据通道部分宽度
- **石头**：较硬，需要持续射击才能击碎
- **铁桶**：被击碎时有额外爆炸特效（纯视觉，不造成伤害）
- **沙袋**：宽扁形状，遮挡面积大
- **铁板**：金属质感，被击中时产生火花特效
- **水泥墙**：大型障碍，需要高火力才能击碎
- **汽车**：被击碎时产生多层爆炸特效
- **装甲车**：极硬，需要高级枪支才能有效击碎
- **钢铁堡垒**：最硬障碍，几乎需要顶级枪支才能击碎

### 4.3 障碍物生成规则

- 同一通道内，相邻障碍物最小间隔 = 障碍物高度 + 40像素
- 左右通道独立生成，但同一时刻屏幕上最多6个障碍物
- 障碍物等级随游戏时间逐步提升（见难度曲线）

## 五、道具系统

### 5.1 枪支道具

- 外观：闪烁的枪支图标 + 等级数字标识
- 生成方式：击碎障碍物后有概率掉落

**掉落概率：**

| 击碎障碍物等级 | 掉落概率 | 可掉落枪支等级 |
|---------------|---------|---------------|
| 1-3级 | 5% | 1-3级 |
| 4-6级 | 8% | 3-6级 |
| 7-9级 | 10% | 6-9级 |
| 10级 | 15% | 8-10级 |

- 掉落的枪支等级在对应范围内随机
- 道具从障碍物消失位置开始下落
- 下落速度与障碍物相同
- 角色触碰道具即拾取，立即替换当前枪支
- 道具到达屏幕底部未拾取则消失（不判定失败）

### 5.2 辅助道具（可选扩展）

| 道具名称 | 效果 | 持续时间 | 掉落概率 |
|---------|------|---------|---------|
| 护盾 | 可抵挡1次障碍物碰撞不死 | 一次性 | 3% |
| 减速时钟 | 所有障碍物下落速度降至40%（不影响玩家和武器） | 6秒 | 3% |
| 双倍积分 | 积分获取翻倍 | 10秒 | 4% |
| 全屏清除 | 清除屏幕上所有障碍物 | 即时 | 1% |

## 六、难度曲线与平衡设计

### 6.1 难度阶段划分

| 阶段 | 游戏时间 | 障碍物等级范围 | 生成间隔(秒) | 下落速度倍率 | 同时最大数 |
|------|---------|---------------|-------------|-------------|-----------|
| 新手期 | 0-30秒 | 1-2 | 2.5 | 1.0x | 3 |
| 热身期 | 30-90秒 | 1-3 | 2.0 | 1.1x | 4 |
| 挑战期 | 90-180秒 | 2-5 | 1.6 | 1.2x | 4 |
| 困难期 | 180-300秒 | 3-7 | 1.3 | 1.4x | 5 |
| 炼狱期 | 300-480秒 | 5-9 | 1.0 | 1.6x | 5 |
| 地狱期 | 480秒+ | 7-10 | 0.8 | 1.8x | 6 |

### 6.2 平衡原则

- 枪支DPS应略高于同等级障碍物的击碎需求，保证能击碎但需要持续射击
- 高级障碍物出现时，应同步提供高级枪支道具的掉落机会
- 每个难度阶段持续约1-2分钟，给玩家适应时间
- 障碍物等级分布为范围内随机，不会连续出现最高等级
- 下落速度倍率确保玩家有足够反应时间

### 6.3 枪支与障碍物平衡对照

| 枪支等级 | 可有效应对的障碍物等级 | 击碎所需时间(约) |
|---------|---------------------|----------------|
| 1级 | 1-2级 | 2-5秒 |
| 2级 | 1-3级 | 1-4秒 |
| 3级 | 2-4级 | 1-3秒 |
| 4级 | 2-5级 | 1-4秒 |
| 5级 | 3-6级 | 1-4秒 |
| 6级 | 4-7级 | 1-4秒 |
| 7级 | 4-8级 | 2-6秒（穿透优势） |
| 8级 | 5-9级 | 1-4秒 |
| 9级 | 6-10级 | 2-8秒（范围优势） |
| 10级 | 7-10级 | 1-3秒 |

> 注：击碎时间假设障碍物从屏幕顶部到底部约5秒行程，持续射击

## 七、视觉与音效设计

### 7.1 像素美术风格

- 分辨率：游戏内逻辑分辨率 240×400，缩放至屏幕
- 色彩：16-bit风格，饱和度偏高，色彩鲜明
- 角色：16×24像素，简洁但有辨识度的像素小人
- 枪支：随等级变化外观，8×8至16×16像素
- 障碍物：各有独特像素外观，受击时闪白2帧
- 背景：深色城市废墟风格，缓慢滚动增加纵深感

### 7.2 粒子特效

| 特效名称 | 触发条件 | 粒子数 | 粒子外观 |
|---------|---------|--------|---------|
| 命中火花 | 子弹击中障碍物 | 3-5 | 小白/黄色火花 |
| 木屑飞溅 | 击中木箱/树干 | 4-6 | 棕色碎片 |
| 石屑飞溅 | 击中石头/水泥墙 | 4-6 | 灰色碎片 |
| 金属火花 | 击中铁桶/铁板/装甲车 | 5-8 | 亮白/蓝色火花 |
| 爆炸碎片 | 障碍物被击碎 | 8-15 | 对应颜色碎片 |
| 爆炸闪光 | 障碍物被击碎 | 1 | 白色闪光圆 |
| 火箭爆炸 | 火箭筒命中 | 15-20 | 橙红色火焰碎片 |
| 激光照射 | 激光炮持续照射 | 持续 | 蓝白色光粒子 |
| 枪口火焰 | 每次射击 | 2-3 | 橙黄色小火焰 |
| 道具闪光 | 枪支道具出现 | 持续 | 彩色闪烁星 |
| 道具拾取 | 角色触碰道具 | 6-8 | 向外扩散星芒 |

### 7.3 音效设计

**枪支音效（每把枪独特的射击声）：**

| 等级 | 名称 | 音效描述 |
|------|------|---------|
| 1 | 左轮手枪 | 低沉的"砰"，回音较长 |
| 2 | 手枪 | 清脆的"啪"，短促 |
| 3 | 双持手枪 | 交替的"啪啪"，有节奏感 |
| 4 | 冲锋枪 | 快速的"哒哒哒哒"，连续感 |
| 5 | 霰弹枪 | 厚重的"轰"，低频 |
| 6 | 步枪 | 有力的"嘭"，中频 |
| 7 | 狙击枪 | 尖锐的"嗖—啪"，穿透感 |
| 8 | 加特林 | 密集的"嗡嗡嗡"，机械感 |
| 9 | 火箭筒 | 巨大的"轰！！"，爆炸感 |
| 10 | 激光炮 | 持续的"嗡——"，电子感 |

**命中音效（根据障碍物材质）：**

| 材质 | 音效 |
|------|------|
| 木箱/树干 | 沉闷的"咚" |
| 石头/水泥墙 | 清脆的"叮" |
| 铁桶/铁板/装甲车 | 金属的"铛" |
| 沙袋 | 柔软的"噗" |
| 汽车 | 金属弯曲的"嘎吱" |
| 钢铁堡垒 | 厚重的"铛"（低频加强） |

**其他音效：**

- 障碍物炸碎：根据材质不同的爆炸声
- 道具掉落：清脆的"叮铃"提示音
- 道具拾取：上升音阶的"叮~叮~"
- 游戏失败：低沉的"嗡——" + 碎裂声
- 连击提示：每达到连击里程碑播放上升音阶
- 背景音乐：快节奏8-bit电子乐，随难度阶段加速

## 九、技术方案

### 9.1 技术选型

| 技术领域 | 选型 | 说明 |
|---------|------|------|
| 游戏引擎 | Three.js 0.164+ | 3D渲染引擎，用于构建像素风格的3D场景 |
| 渲染方式 | WebGL (Three.js WebGLRenderer) | 硬件加速渲染，保证60fps |
| 像素风格 | 正交相机 + nearest-neighbor纹理过滤 | 实现像素锐利的视觉效果 |
| 音频系统 | Web Audio API | 程序化生成音效，无需加载音频文件 |
| 输入处理 | 鼠标事件 + 触屏事件 | 统一的点击切换通道交互 |
| 模块系统 | ES Modules (import/export) | 原生模块化，无需打包工具即可开发 |
| 数据存储 | localStorage | 保存最高分和游戏设置 |
| 构建工具 | 简易Node.js脚本 | 仅用于生产环境打包，开发阶段零构建 |

### 9.2 参考项目架构

本项目参考 `xuancaijiezou3D-main`（Three.js 弹跳球游戏）的架构模式，采用相同的模块化设计思路：

**参考项目核心架构：**


**参考项目关键技术模式：**

1. **集中式状态管理**：`state.js` 导出全局状态对象，所有模块共享
2. **输入策略模式**：`input.js` 定义 `mouseInputStrategy` 和 `touchInputStrategy`，运行时自动检测并切换
3. **程序化音效**：`audio.js` 使用 `OfflineAudioContext` 预渲染音效Buffer，运行时通过 `AudioBufferSourceNode` 播放，零音频文件依赖
4. **Shader粒子系统**：`effects.js` 使用 `ShaderMaterial` + `BufferGeometry` 实现高性能粒子特效
5. **固定时间步物理**：`game.js` 的 `animate()` 使用 `FIXED_DT = 1/120` 进行物理模拟，确保帧率无关的物理行为
6. **移动端适配**：`scene.js` 通过 `isMobile()` 检测，动态调整渲染质量（抗锯齿、阴影、粒子数）
7. **ES Module importmap**：`index.html` 使用 `<script type="importmap">` 映射 Three.js CDN，开发阶段零构建

```javascript
const clickInputStrategy = {
  init() {
    // PC: 鼠标点击
    this._onClick = (e) => {
      if (!state.gameActive) return;
      switchChannel();
    };
    document.addEventListener('click', this._onClick);

    // 移动端: 触屏点击
    this._onTouchStart = (e) => {
      if (!state.gameActive) return;
      e.preventDefault();
      switchChannel();
    };
    const canvas = state.renderer.domElement;
    canvas.addEventListener('touchstart', this._onTouchStart, { passive: false });
  },

  destroy() {
    document.removeEventListener('click', this._onClick);
    const canvas = state.renderer.domElement;
    if (canvas) canvas.removeEventListener('touchstart', this._onTouchStart);
  },

  onGameStart() {
    state.gameActive = true;
  },

  onGameEnd() {
    state.gameActive = false;
  }
};

function switchChannel() {
  state.currentChannel = state.currentChannel === 'left' ? 'right' : 'left';
  state.playerTargetX = state.currentChannel === 'left'
    ? -CHANNEL_WIDTH / 2
    : CHANNEL_WIDTH / 2;
```

#### 9.4.4 音效系统

```javascript
// audio.js - 参考项目的程序化音效方案
// 每种枪支预渲染独特的射击音效Buffer
// 每种障碍物材质预渲染独特的命中音效Buffer

async function renderGunShotBuffer(gunConfig) {
  const { frequency, type, attack, decay, sustain, duration, release, harmonics } = gunConfig.audio;
  const totalDuration = attack + decay + duration + release + 0.05;
  const sampleRate = audioCtx.sampleRate;
  const length = Math.ceil(totalDuration * sampleRate);
  const offline = new OfflineAudioContext(1, length, sampleRate);

  const osc = offline.createOscillator();
  const gain = offline.createGain();
  const filter = offline.createBiquadFilter();

  osc.type = type;
  osc.frequency.value = frequency;
  filter.type = 'lowpass';
  filter.frequency.value = type === 'sine' ? 2000 : 1200;

  // ADSR envelope
  applyADSR(gain, 0, 0.35, attack, decay, sustain, duration, release);

  osc.connect(filter);
  filter.connect(gain);
  gain.connect(offline.destination);
  osc.start(0);
  osc.stop(totalDuration);

  return await offline.startRendering();
}
```

#### 9.4.5 粒子特效

```javascript
// effects.js - 参考项目的Shader粒子方案
// 使用BufferGeometry + ShaderMaterial实现高性能粒子

function createHitParticles(position, color, count) {
  const geo = new THREE.BufferGeometry();
  const posArr = new Float32Array(count * 3);
  const velArr = [];  // 存储速度，JS端计算

  for (let i = 0; i < count; i++) {
    posArr[i * 3] = position.x;
    posArr[i * 3 + 1] = position.y;
    posArr[i * 3 + 2] = position.z;
    const angle = Math.random() * Math.PI * 2;
    const speed = 1 + Math.random() * 3;
    velArr.push({ x: Math.cos(angle) * speed, y: Math.sin(angle) * speed });
  }

  geo.setAttribute('position', new THREE.BufferAttribute(posArr, 3));

  const mat = new THREE.PointsMaterial({
    color: color,
    size: 2,
    transparent: true,
    opacity: 1.0,
    blending: THREE.AdditiveBlending,
    depthWrite: false
  });

  return { mesh: new THREE.Points(geo, mat), velocities: velArr, life: 0.5 };
}
```

#### 9.4.6 游戏主循环

```javascript
// game.js - 参考项目的固定时间步物理
function animate() {
  requestAnimationFrame(animate);
  const now = performance.now() / 1000;
  let dt = now - state.lastTime;
  state.lastTime = now;
  dt = Math.min(dt, 0.05);

  // 固定时间步物理模拟
  const FIXED_DT = 1 / 120;
  const maxSteps = state.isMobile ? 4 : 2;
  const physicsSteps = Math.min(maxSteps, Math.max(1, Math.round(dt / FIXED_DT)));
  const physicsDt = dt / physicsSteps;

  for (let step = 0; step < physicsSteps; step++) {
    updatePlayerPosition(physicsDt);   // 角色通道切换动画
    updateBullets(physicsDt);          // 子弹移动
    updateObstacles(physicsDt);        // 障碍物下落
    checkBulletCollisions();           // 子弹-障碍物碰撞
    checkPlayerCollisions();           // 角色-障碍物碰撞
    updateItems(physicsDt);            // 道具下落
    checkItemPickup();                 // 道具拾取
  }

  updateEffects(dt);                   // 粒子特效（不依赖物理步）
  updateDifficulty(dt);                // 难度递增
  updateUI();                          // HUD更新
  state.renderer.render(state.scene, state.camera);
}
```

### 9.5 性能优化策略

| 优化项 | 策略 | 参考 |
|--------|------|------|
| 渲染分辨率 | 低分辨率渲染(240×400) + CSS pixelated缩放 | 像素风格天然节省GPU |
| 移动端适配 | 检测移动设备，降低粒子数、关闭阴影 | 参考项目 `isMobile()` 模式 |
| 对象池 | 子弹、粒子使用对象池复用，避免GC | 参考项目 `activePool` 模式 |
| 音效池 | 限制同时播放音效数量，自动淘汰最旧 | 参考项目 `MAX_CONCURRENT = 4` |
| 纹理过滤 | 全局使用 NearestFilter | 像素风格核心 |
| 几何体复用 | 同等级障碍物共享几何体和材质 | 减少GPU draw call |

## 十二、V2.5 — 渐进升级 + 道具扩展 + 波次系统

### 12.1 问题分析

#### 当前系统痛点

| 问题 | 现状 | 影响 |
|------|------|------|
| 障碍物跳跃升级 | `DIFFICULTY_STAGES` 用 `levelMin/levelMax` 范围随机选取，如"困难期"直接从level 2跳到level 5 | 玩家突然面对高HP障碍物，武器伤害跟不上 |
| 武器跳跃升级 | `tryDropItem` 在 `itemLevelMin~itemLevelMax` 范围随机，可能从左轮手枪直接跳到霰弹枪 | 跳过中间武器体验，玩家无适应期 |
| 道具无名称 | `createItem`/`createPowerup` 只显示彩色方块+等级条，无文字标识 | 玩家无法识别道具类型，拾取决策盲目 |
| 纯随机生成 | 障碍物/道具/通道全部随机，不可控因素高 | 体验不稳定，可能连续无道具或连续同通道 |
| 缺少临时武器 | 只有永久武器升级，无短期强力体验 | 缺少"爆发时刻"的游戏乐趣 |
| 缺少分身道具 | 只有防御/辅助类道具，无进攻型扩展 | 玩法单一 |

### 12.2 功能设计

#### 12.2.1 渐进式障碍物升级

**核心思路**：追踪玩家已解锁的最高障碍物等级，新障碍物只在当前等级±1范围内生成。

**实现方案**：

1. 新增 `state.maxObstacleLevel`（当前已解锁的最高障碍物等级，初始=0）
2. 修改 `spawnObstacle()` 逻辑：
   - 生成等级 = `maxObstacleLevel` 或 `maxObstacleLevel - 1`（70%/30%概率）
   - 当玩家累计摧毁 N 个当前最高等级障碍物后，`maxObstacleLevel++`
   - 解锁阈值：`destroyedAtCurrentLevel >= 3 + maxObstacleLevel * 2`
3. 移除 `DIFFICULTY_STAGES` 中的 `levelMin/levelMax`，改用 `maxObstacleLevel` 动态控制
4. 保留 `speedMult`/`spawnInterval`/`maxObstacles` 随时间递增（这些与障碍物等级无关）

**修改文件**：`config.js`、`obstacles.js`、`state.js`

#### 12.2.2 渐进式武器升级

**核心思路**：武器道具只掉落比当前武器高1级的版本，确保玩家逐步体验每把武器。

**实现方案**：

1. 修改 `tryDropItem()`：
   - 武器掉落等级 = `state.currentWeaponIndex + 1`（固定+1，不再随机范围）
   - 如果 `+1` 超过当前难度允许的最大武器等级，则不掉落
   - 最大允许武器等级由 `state.maxObstacleLevel` 决定（武器等级 ≤ maxObstacleLevel + 1）
2. 移除 `DIFFICULTY_STAGES` 中的 `itemLevelMin/itemLevelMax`
3. 新增 `state.maxWeaponDropLevel` 控制可掉落的武器上限

**修改文件**：`items.js`、`config.js`、`state.js`

#### 12.2.3 道具名称显示

**核心思路**：在道具方块下方添加像素文字标签，使用 Canvas2D 渲染文字纹理贴到 Three.js PlaneGeometry 上。

**实现方案**：

1. 新增 `js/texttexture.js` 工具模块：
   - `createTextTexture(text, color, fontSize)` → 返回 `{ texture, width, height }`
   - 使用 OffscreenCanvas/Canvas 渲染文字，生成 Three.js CanvasTexture
   - 设置 `texture.magFilter = NearestFilter` 保持像素风格
   - 缓存机制：相同文字+颜色复用同一纹理

2. 修改 `createItem()`（items.js）：
   - 在道具方块下方添加名称标签 PlaneGeometry
   - 使用 `createTextTexture(weapon.name, weapon.bulletColor)` 生成纹理
   - 标签位置：`y = -12, z = 11`

3. 修改 `createPowerup()`（items.js）：
   - 在道具方块下方添加名称标签
   - 使用 `createTextTexture(config.name, config.color)` 生成纹理

4. 清理：道具被移除时 dispose 纹理和几何体

**新增文件**：`js/texttexture.js`
**修改文件**：`items.js`

#### 12.2.4 新辅助道具：临时武器

**配置**：
```javascript
{ type: 'tempweapon', name: '临时武器', color: 0xFF6600, duration: 20, icon: 'W' }
```

**机制**：
- 拾取后，保存当前武器等级到 `state.savedWeaponIndex`
- 将 `state.currentWeaponIndex` 设为 `savedWeaponIndex + 3`（上限9，即加特林/火箭筒级别）
- 更新枪械外观，显示临时武器特效（枪口橙色闪光）
- 20秒后自动恢复为 `savedWeaponIndex` 的武器
- 临时武器期间拾取武器道具：升级 `savedWeaponIndex`（而非 currentWeaponIndex），到期后获得升级后的武器
- HUD显示：`🔫临时武器 15.2s`

**实现方案**：

1. `config.js`：POWERUPS 数组添加 `tempweapon` 配置
2. `state.js`：添加 `tempWeaponActive`、`tempWeaponTimer`、`savedWeaponIndex`
3. `items.js`：
   - `activatePowerup()` 添加 `tempweapon` 分支
   - `updatePowerupTimers()` 添加临时武器倒计时逻辑
   - 到期恢复逻辑：`currentWeaponIndex = savedWeaponIndex`，更新枪外观
   - `pickupItem()` 修改：临时武器激活时，武器升级应用到 `savedWeaponIndex`
4. `weapons.js`：临时武器期间子弹颜色使用临时武器颜色（0xFF6600 橙色）
5. `ui.js`：HUD 显示临时武器倒计时

**修改文件**：`config.js`、`state.js`、`items.js`、`weapons.js`、`ui.js`

#### 12.2.5 新辅助道具：分身

**配置**：
```javascript
{ type: 'clone', name: '分身', color: 0x00FFAA, duration: 20, icon: 'C' }
```

**机制**：
- 拾取后，在对侧通道创建一个半透明克隆玩家
- 分身始终与玩家在对侧通道（玩家左则分身右，反之亦然）
- 分身独立开火，使用与玩家相同的武器
- 分身子弹独立追踪，碰撞检测与主子弹相同
- 分身击杀障碍物同样计分/计连击
- 20秒后分身消失
- HUD显示：`👥分身 15.2s`

**实现方案**：

1. `config.js`：POWERUPS 数组添加 `clone` 配置
2. `state.js`：添加 `cloneActive`、`cloneTimer`、`cloneGroup`、`cloneBullets`
3. `player.js`：
   - 新增 `createClone()`：创建半透明（opacity 0.6）的玩家副本 Group
   - 新增 `updateClone()`：分身位置始终在玩家对侧通道
   - 新增 `removeClone()`：移除分身 Group，dispose 资源
4. `weapons.js`：
   - `updateWeapons()` 中，如果分身激活，同时发射分身子弹
   - 分身子弹使用不同 z 层（z=9）或不同颜色（半透明）以区分
   - `updateBullets()` 中同时更新分身子弹
5. `items.js`：
   - `activatePowerup()` 添加 `clone` 分支
   - `updatePowerupTimers()` 添加分身倒计时
   - 到期调用 `removeClone()`
6. `game.js`：gameLoop 中更新分身位置和子弹
7. `ui.js`：HUD 显示分身倒计时

**修改文件**：`config.js`、`state.js`、`player.js`、`weapons.js`、`items.js`、`game.js`、`ui.js`

#### 12.2.6 波次关卡系统

**核心思路**：用预生成的波次数据替代纯随机生成，在固定框架内增加小范围随机性。

**数据结构**：

```javascript
// 波次条目
WaveEntry = {
  time: number,          // 该波开始时间（秒）
  duration: number,      // 波次持续时间
  obstacleLevel: number, // 基础障碍物等级
  spawnPattern: string,  // 生成模式: 'single'|'double'|'alternating'|'burst'
  spawnInterval: number, // 生成间隔（秒）
  speedMult: number,     // 速度倍率
  maxObstacles: number,  // 同屏最大障碍物数
  guaranteedDrop: {      // 保证掉落（每波一次）
    type: 'weapon' | 'powerup',
    weaponLevel?: number,
    powerupType?: string,
    time: number,        // 波次内第几秒掉落
  } | null,
  calmAfter: number,     // 波次结束后的平静期（秒）
}
```

**波次生成算法** (`generateWavePlan()`)：

```
波次1-3:   教学波，obstacleLevel=0, 同通道, 保证武器升级到level 1
波次4-6:   入门波，obstacleLevel=0-1, 双通道, 保证武器升级到level 2
波次7-9:   进阶波，obstacleLevel=1-2, 双通道, 保证武器升级到level 3
波次10-12: 挑战波，obstacleLevel=2-3, 双通道, 保证武器升级到level 4, 首次道具掉落
波次13-15: 困难波，obstacleLevel=3-4, burst模式, 保证武器升级到level 5
波次16-18: 炼狱波，obstacleLevel=5-6, double模式, 保证武器升级到level 6-7
波次19+:   地狱波，obstacleLevel=7+, 全模式, 武器升级到level 8-10
```

**随机性控制**：
- 生成间隔：`spawnInterval ± 15%`
- 通道选择：70%按pattern，30%随机
- 障碍物等级：`obstacleLevel ± 1`（20%概率）
- 道具掉落：保证掉落 + 额外 5% 随机掉落

**波次间过渡**：
- 每波结束后 2-3 秒平静期（无障碍物生成）
- 平静期显示波次提示："WAVE X" 像素文字

**实现方案**：

1. 新增 `js/waves.js` 波次生成模块：
   - `generateWavePlan()` → 返回 WaveEntry[]
   - `getCurrentWave(gameTime)` → 返回当前波次
   - `getWaveSpawnConfig(gameTime)` → 返回当前生成配置（含随机扰动）

2. 修改 `obstacles.js`：
   - `spawnObstacle()` 使用波次配置替代 `DIFFICULTY_STAGES`
   - 生成等级由波次 `obstacleLevel` 决定（替代 `levelMin/levelMax`）

3. 修改 `items.js`：
   - `tryDropItem()` 使用波次的 `guaranteedDrop` 配置
   - 武器掉落等级由波次保证 +1 渐进

4. 修改 `game.js`：
   - `startGame()` 调用 `generateWavePlan()` 生成波次计划
   - `gameLoop()` 使用波次系统驱动障碍物生成和道具掉落
   - 波次切换时显示 "WAVE X" 提示

5. 修改 `config.js`：
   - 保留 `DIFFICULTY_STAGES` 作为速度/密度参考（向后兼容）
   - 移除 `levelMin/levelMax`/`itemLevelMin/itemLevelMax` 字段

6. 修改 `state.js`：
   - 添加 `wavePlan`、`currentWaveIndex`、`waveTimer`、`calmTimer`
   - 添加 `maxObstacleLevel`、`destroyedAtCurrentLevel`

7. 修改 `effects.js`：
   - 新增 `spawnWaveAnnouncement(waveIndex)` 波次提示特效

**新增文件**：`js/waves.js`
**修改文件**：`config.js`、`state.js`、`obstacles.js`、`items.js`、`game.js`、`effects.js`

### 12.5 技术风险与注意事项

| 风险 | 应对策略 |
|------|---------|
| Canvas2D 文字纹理在低分辨率下模糊 | 使用 2x 分辨率渲染 + NearestFilter 缩放 |
| 分身子弹数量翻倍导致性能问题 | 分身子弹共享 MAX_BULLETS 上限，或分身使用更低射速 |
| 波次系统与现有难度曲线冲突 | 波次系统完全替代 DIFFICULTY_STAGES 的等级控制，保留速度/密度控制 |
| 临时武器到期恢复时枪外观闪烁 | 到期前 3 秒枪口闪烁预警，恢复时平滑过渡 |
| 分身与主玩家通道切换时序 | 分身位置在 updatePlayer() 中同步更新，确保始终在对侧 |
| 纹理缓存内存占用 | 限制缓存大小（最多 20 个纹理），LRU 淘汰 |
